# Платформа корпоративного навчання — RESTful API

Серверна частина (Backend) для системи управління корпоративним навчаннями (LMS). Додаток реалізує функції адміністрування навчальних курсів, категоризації напрямків підготовки, автентифікації користувачів із розмежуванням прав доступу на основі ролей (RBAC), а також асинхронного фонового оброблення подій.

## Технологічний стек
* Мова програмування: Java 21 (JDK 21)
* Фреймворк: Spring Boot 3.x (Spring Data JPA, Spring Security)
* База даних: PostgreSQL 16
* Керування міграціями БД: Flyway
* Брокер повідомлень: RabbitMQ (AMQP протокол)
* Безпека: Безсесійна автентифікація за допомогою JWT (JSON Web Tokens)
* Контейнеризація: Docker, Docker Compose
* Тестування: JUnit 5, MockMvc, Testcontainers (PostgreSQL)

## Архітектурні особливості
1. **Рольова модель доступу:** Додаток підтримує дві ролі користувачів: `USER` (Студент) та `ADMIN` (Інструктор/Менеджер). Операції створення, модифікації та видалення навчальних курсів захищені на рівні методів контролера та доступні виключно для адміністраторів додатка.
2. **Асинхронні сповіщення:** При публікації нового курсу бізнес-шар платформи генерує подію, яка через RabbitTemplate відправляється у стійку чергу подій. Окремий фоновий компонент-слухач перехоплює повідомлення для імітації розсилки поштових сповіщень співробітникам.
3. **Стабільність підключень:** Конфігурація Docker Compose містить блоки перевірки працездатності (healthcheck) для інфраструктурних контейнерів баз даних та брокера, що гарантує старт веб-додатка лише після повної готовності суміжних сервісів.

## Інструкція із розгортання та запуска

Програмний продукт підтримує два режими розгортання: автоматичний (за допомогою інсталяційного командного сценарію) та покроковий ручний.

### Варіант 1. Автоматизований запуск (в 1 клік)
Для швидкого налаштування середовища, компіляції артефакту та запуску всього мікросервісного кластера скористайтеся готовим інсталяційним скриптом у кореневому каталозі проєкту.

Для Windows:
```bash
.\run.cmd

```

### Варіант 2. Ручний покроковий запуск

**1. Збірка проєктного артефакту**
Для компіляції вихідного коду та пакування додатка у виконуваний JAR-файл виконайте команду в кореневому каталозі проєкту:

Для Windows:

```bash
.\mvnw clean package -DskipTests

```

Для Linux / macOS:

```bash
./mvnw clean package -DskipTests

```

**2. Оркестрація та запуск у Docker контейнерах**
Після успішної збірки запустіть розгортання кластера через Docker Compose:

```bash
docker compose up --build

```

Після ініціалізації середовища та успішного застосування міграцій Flyway сервер буде доступний за локальною адресою: `http://localhost:8080`.

## Специфікація основних HTTP-ендпоінтів API

### 1. Сервіс автентифікації (Доступ відкритий)

* **Реєстрація користувача / інструктора:**
`POST /api/auth/register`
Тіло запиту (JSON):

```json
{
  "email": "instructor_name@lms.com",
  "password": "securepassword123",
  "role": "ADMIN"
}

```

* **Автентифікація та отримання JWT-токена:**
`POST /api/auth/login`
Тіло запиту (JSON):

```json
{
  "email": "instructor_name@lms.com",
  "password": "securepassword123"
}

```

*Отриманий у відповідь токен необхідно вставляти в HTTP-заголовок кожного наступного запиту: `Authorization: Bearer <ваш_токен>`.*

### 2. Управління категоріями (Потрібен токен авторизації)

* **Отримання переліку категорій:** `GET /api/categories`
* **Створення нової категорії:** `POST /api/categories`
Тіло запиту (JSON):

```json
{
  "name": "Інженерія програмного забезпечення"
}

```

### 3. Управління курсами (Потрібен токен авторизації)

* **Отримання курсів з фільтрацією та пагінацією:** `GET /api/courses?search=Spring&page=0&size=10`
* **Створення курсів (Доступно тільки для ролі ADMIN):** `POST /api/courses`
Тіло запиту (JSON):

```json
{
  "title": "Розробка архітектури на Spring Boot 3",
  "instructor": "Name Surname",
  "durationHours": 48,
  "categoryId": 1
}

```

* **Видалення курсу з платформи (Доступно тільки для ролі ADMIN):** `DELETE /api/courses/{id}`
